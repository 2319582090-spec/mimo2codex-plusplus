# 项目描述（可直接粘贴到比赛平台）

## 项目名称
mimo2codex++ — 多账号 MiMo 网关 + 额度看板

## 一句话简介
通过多账号并发突破小米 MiMo API 的单账号并发上限，彻底解决 429 限流问题。

## 问题
小米 MiMo 模型的 API 有严格的并发限制（Standard 套餐 20 并发、500 RPM、100K TPM）。超过限制就返回 429 错误，请求直接失败。对于需要高并发的开发者（Agent 批量调用、自动化工作流），这是硬瓶颈。

## 解决方案
mimo2codex++ 是一个本地代理网关，通过多账号池化管理，把 N 个账号的并发能力叠加：
- 2 个 Standard 账号 = 40 并发、1000 RPM、200K TPM
- 自动 429 避让：一个账号被限流，自动切换到其他账号
- 额度看板：实时看到每个账号的使用情况和健康状态

## 技术实现
1. **Pool Manager**：每个 Key 对应一个 mimo2codex 实例，智能负载均衡
2. **429 避让**：遇到 429 自动进入 30 秒冷却期，自动切换到其他账号
3. **健康检查**：每 60 秒检查一次实例状态，自动重启崩溃的实例
4. **额度查询**：实时查询每个 Key 的额度使用情况
5. **安全存储**：API Key 使用 AES-256-GCM 加密，仅存储在本地
6. **一键配置**：自动生成 Codex 配置文件，一键复制

## 创新点
- 不是另一个单 Key 适配器，而是真正的多账号编排系统
- 429 避让 + 冷却恢复，无需手动干预
- 额度可视化，让多账号管理变得透明
- 开箱即用，一行命令启动
