# 演示脚本（2-3 分钟）

## 开场（20秒）
"大家好，我是 SSCQ。今天演示的是 mimo2codex++，一个解决小米 MiMo API 并发限制的工具。"

## 问题演示（40秒）
"小米 MiMo 模型的 API 有严格的并发限制。Standard 套餐只有 20 并发。"
"当我用 Agent 批量调用时，很快就触发 429 错误——请求直接失败。"
"单账号的并发上限是硬瓶颈，没法绕过。"

## 解决方案演示（60秒）
"mimo2codex++ 通过多账号池化管理来解决这个问题。"
"我有 3 个 Standard 套餐账号，每个 20 并发。"
"mimo2codex++ 把它们叠加起来——60 并发、1500 RPM、300K TPM。"
"而且，它会自动避让 429：一个账号被限流，请求自动路由到其他账号。"
"Dashboard 上可以实时看到每个账号的健康状态和额度使用情况。"

## 技术亮点（30秒）
"技术上，Pool Manager 实现了智能负载均衡——健康优先、最低在途请求、最少总请求。"
"遇到 429 自动进入 30 秒冷却期，冷却结束后自动恢复。"
"API Key 使用 AES-256-GCM 加密，仅存储在本地。"

## 结尾（10秒）
"mimo2codex++，让多账号并发不再受限。仓库地址在描述区，欢迎 star。谢谢！"
